# Milestones

Milestone definitions and sprint-to-milestone mapping for the NextLevelApex backlog.

## Milestone Definitions

| Milestone | Name | Description |
|-----------|------|-------------|
| M1 | Core Operator MVP | Functional CLI orchestrator + read-only GUI with diagnose/dry-run, history, compare, export/import, tooltips, and dev setup. |
| M2 | Visual & UX Polish | Design system lock, theming, transitions, ambient effects, live output streaming, and navigation IA. |
| M3 | Hardening & QA | Performance tuning, accessibility hardening, URL sync fixes, operator-grade QA megasuite, release cert v2, and worktree guardrails v2. |
| M4 | Governance & Safety | Definition of Done enforcement, stop conditions, and the Operator Execution Safety System. |

## Mapping Rules

1. Each sprint belongs to exactly one milestone.
2. Milestones are ordered: M1 sprints ship before M2, M2 before M3, etc.
3. Within a milestone, sprint order is advisory (not enforced), but dependency chains must be respected.
4. A milestone is complete when all its sprints have status `done`.
